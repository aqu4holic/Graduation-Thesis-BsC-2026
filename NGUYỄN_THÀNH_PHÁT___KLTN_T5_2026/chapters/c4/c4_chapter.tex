% Balanced Accuracy được tính như sau:
% \begin{equation}
%     \mathrm{BA}_{\mathrm{local}} = \frac{1}{C}\sum_{c=1}^{C}

\chapter{Experimental Setup and Evaluation}
\label{chap:experiments}

This chapter describes the complete experimental setup: the dataset, hardware and software environment, local evaluation procedure, CrunchDAO submission workflow, and the comparative experiments between the baseline and proposed improvements from Chapter~\ref{chap:method}.

% =====================================================================
\section{Dataset}
\label{sec:dataset}
% =====================================================================

\subsection{Data Source}

All experiments use the official dataset from the ADIA Lab Causal Discovery Challenge~\cite{crunchdao_adia_docs, crunchdao_causal_2024}. According to the official report by the organizers~\cite{olivetti2025adialab}, the competition uses a total of 47,000 datasets split into approximately 25,000 training datasets and 22,000 test datasets (further divided into public and private test sets). Through the CrunchDAO platform, this thesis accesses a training set of \textbf{23,500 dataset instances} and a test set of \textbf{1,880 dataset instances} (public test set). Neither set comes with direct class labels; instead, each dataset instance is accompanied by an \textbf{adjacency matrix} $\mathbf{A} \in \{0,1\}^{p \times p}$ representing the ground-truth causal DAG, where $\mathbf{A}_{ij} = 1$ if and only if a directed edge $i \to j$ exists.

\subsection{Label Derivation from Adjacency Matrix}
\label{subsec:label_derivation}

The 8-class label for each variable $K$ is derived from the adjacency matrix $\mathbf{A}$ by analyzing the local structure of $K$ in the graph with respect to the pair $(X, Y)$. Specifically, for each variable $K \neq X, K \neq Y$, four bits are checked: $\mathbf{A}_{K,X}$ (edge $K \to X$), $\mathbf{A}_{X,K}$ (edge $X \to K$), $\mathbf{A}_{K,Y}$ (edge $K \to Y$), and $\mathbf{A}_{Y,K}$ (edge $Y \to K$). The class label of $K$ is determined from these four bits according to Table~\ref{tab:label_rules}:

\begin{table}[h]
\centering
\caption{Label derivation rules from the adjacency matrix for variable $K$.}
\label{tab:label_rules}
\begin{tabular}{lcccc}
\hline
\textbf{Label} & $K{\to}X$ & $X{\to}K$ & $K{\to}Y$ & $Y{\to}K$ \\
\hline
Confounder       & 1 & 0 & 1 & 0 \\
Mediator         & 0 & 1 & 1 & 0 \\
Collider         & 0 & 1 & 0 & 1 \\
Independent      & 0 & 0 & 0 & 0 \\
Cause of X       & 1 & 0 & 0 & 0 \\
Consequence of X & 0 & 1 & 0 & 0 \\
Cause of Y       & 0 & 0 & 1 & 0 \\
Consequence of Y & 0 & 0 & 0 & 1 \\
\hline
\end{tabular}
\end{table}

Label derivation is performed once during preprocessing and cached for use during training. For the test set (1,880 instances), the ground-truth adjacency matrices are not publicly available -- the CrunchDAO system derives labels internally when scoring submissions.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.6\linewidth]{figures/label_derivation.png}
    \caption{Label derivation pipeline from adjacency matrix $\mathbf{A}$: for each variable $K$, check the four relevant edges to $X$ and $Y$, then look up the corresponding class in the rule table.}
    \label{fig:label_derivation}
\end{figure}

\subsection{Dataset Structure}

Each \textit{dataset instance} consists of two components: (1) an observation matrix $\mathbf{D} \in \mathbb{R}^{1000 \times p}$ with $p \in \{3,\ldots,10\}$ variables, and (2) an adjacency matrix $\mathbf{A} \in \{0,1\}^{p \times p}$ of the ground-truth causal DAG. The two variables $X$ and $Y$ are pre-identified in the metadata, with $\mathbf{A}_{X,Y} = 1$ by assumption. The total number of $(K, \text{label})$ pairs in the training set exceeds 23,500 since each dataset with $p > 3$ contributes more than one variable $K$.

\subsection{Synthetic Data Generation}

According to~\cite{olivetti2025adialab}, data is generated from SCMs with nonlinear causal relationships (neural networks, Gaussian Processes, Bayesian models) over six types of synthetic graphs (Hierarchical, Bipartite, Erd\H{o}s-R\'enyi, Low-rank, Scale-free, Preferential Attachment) and real-world food-web graphs. Output is normalized to $[-1, +1]$ to prevent leaking structural information through variance. Post-competition analysis shows Hierarchical graphs are the easiest to classify, while Preferential Attachment is consistently the hardest across all solutions~\cite{olivetti2025adialab}.

\subsection{Label Distribution Characteristics}

The eight classes are not uniformly distributed in the training set. Classes such as Independent, Cause of X, and Consequence of Y tend to be more frequent, while Mediator and Collider have significantly fewer samples. This imbalance directly motivates the use of inverse-frequency class weights in the loss function (~\ref{sec:loss}) and Balanced Accuracy as the evaluation metric.

\section{Experimental Environment}
\label{sec:environment}
% =====================================================================

\subsection{Hardware}

All experiments are conducted on a server with the following configuration: two NVIDIA RTX 5880 GPUs (48GB combined VRAM), a 48-core CPU for parallelizing edge tensor preprocessing, and 251 GB RAM to hold the full preprocessed training set in memory. Early experiments (v2 through v8b base) run on a single A100 40GB GPU; experiments on the XY augmentation set ($\sim\!263$K samples) run on two RTX 5880 GPUs using DDP (Distributed Data Parallel) as the dataset size exceeds single-GPU memory. The dual-path transformer (v4) required an A100 80GB due to OOM on the 40GB configuration with $p=10$ graphs. Edge tensor preprocessing is parallelized across CPU cores via \texttt{ProcessPoolExecutor}; model training runs on GPU.

\subsection{Software and Libraries}

The programming environment uses Python with the following key libraries: PyTorch and PyTorch Lightning for model construction and training, NumPy and SciPy for preprocessing and statistical computation (including kernel regression and ANM residuals), NetworkX for graph operations during label derivation and augmentation, and the CrunchDAO client library for data access and submission. Key experiment files are: \texttt{v8b\_anm.py} (v8b base), \texttt{v8b\_anm\_xyaug.py} (v8b+XY aug), \texttt{v11\_structbias.py} (v11 base), \texttt{v11\_xyaug.py} (v11+XY aug), \texttt{v13\_fullstack.py} (ML pipeline); the local evaluation script is \texttt{evaluate.py}.

\section{Evaluation Procedure}
\label{sec:evaluation}
% =====================================================================

\subsection{Local Evaluation}
\label{subsec:local_eval}

To rapidly evaluate architectural changes and hyperparameters without submitting to CrunchDAO, the thesis builds a local evaluation pipeline based on a local test set ($y\_\text{test\_reduced}$) stored as a pickle file. The script \texttt{evaluate.py} computes Balanced Accuracy according to Equation~\eqref{eq:balanced_acc} on this set, allowing quick comparison across experimental runs.

Balanced Accuracy is computed as:
\begin{equation}
    \mathrm{BA}_{\mathrm{local}} = \frac{1}{C}\sum_{c=1}^{C}
    \frac{\sum_{i} \mathbb{1}[\hat{y}_i = c \wedge y_i = c]}
         {\sum_{i} \mathbb{1}[y_i = c]},
    \label{eq:local_ba}
\end{equation}
i.e., the mean recall across all $C = 8$ classes. Balanced Accuracy is not affected by class imbalance: a class with 100 samples and one with 1000 samples contribute equally to the score.

The local evaluation pipeline serves as the primary development loop: change $\to$ train $\to$ evaluate local $\to$ decide whether to submit. Only configurations that show significant local score improvement over the baseline are submitted to CrunchDAO.

\subsection{CrunchDAO Public Leaderboard Evaluation}
\label{subsec:crunchdao_submit}

The CrunchDAO platform~\cite{crunchdao_competition} provides a sandboxed model execution environment: users submit a Python notebook (not a predictions file), and the system automatically runs the notebook on the hidden test set to compute Balanced Accuracy. To be compatible with the sandbox, the notebook uses the special pragma \texttt{\# @crunch/keep:on} to mark cells that retain global variables (including model weights and preprocessed data) between execution steps, avoiding recomputation from scratch and staying within the runtime limit. Additionally, an \texttt{IS\_CLOUD\_SUBMIT} flag is used to disable disk caching (setting \texttt{cache\_dir=None}) during inference, since the cloud sandbox has no persistent storage between runs.

In experiments, there is a small gap between local and public scores due to slightly different test set distributions: the baseline achieves \textbf{73.96\%} local but \textbf{73.61\%} public. This indicates that local evaluation is a good but imperfect proxy, and one must be careful not to overfit to local scores.

\subsection{Error Analysis}

Beyond aggregate numbers, the thesis analyzes confusion matrices to understand the most common misclassification errors in the baseline. Representative examples: Mediator is frequently confused with Consequence of X (both have $X \to K$); Collider is frequently confused with Independent (both have marginal $X \perp K$); and Confounder is frequently confused with Cause of X (both have $K \to X$).

With v8b + XY aug, the rate of Collider $\to$ Independent misclassification drops substantially, confirming that ANM residual channels help the model detect the conditional dependence $X \not\perp K \mid Y$ characteristic of Colliders. With v11+, analysis of 139 Mediator error samples shows that ${\sim}57\%$ of errors are due to the model detecting only one half of the chain $X \to K \to Y$, suggesting that X/Y-aware chain bias could further improve Mediator classification.

\subsection{Architectural Experiment Analysis (Negative Results)}

All pure architectural modifications (v3, v3+, v4, v5, v6) yield results at or below the baseline. The cross-attention branch (v3+) drops $1.90\%$; the dual-path transformer (v4) drops $1.19\%$ locally but drops $4.01\%$ publicly due to severe overfitting. These results are consistent with the observation in~\cite{olivetti2025adialab} that \textit{feature engineering and representation learning} are the keys -- not increased architectural complexity. The v13 ML fullstack experiment confirms this further: despite using 300+ engineered features, four causal discovery algorithms, three gradient boosting models, a GNN refinement stage, and a stacking ensemble, it achieves only 72.64\% -- significantly below the simple Conv1D baseline. This confirms that the Conv1D architecture's ability to learn from nonlinear functional curves (the sorted+kernel channels) cannot be matched by scalar feature aggregation, regardless of the sophistication of the feature engineering pipeline.

However, v11 (structural attention bias) is an important exception: the only architectural change yielding a significant positive result ($+5.63\%$ local over v2, $+2.65\%$ over v8b with the same input channels). The key distinction from v3+/v4 is that v11 adds \textit{topologically-motivated causal inductive bias} rather than raw capacity, with minimal additional parameters. Chapter~\ref{chap:results} analyzes the mechanism and significance of this distinction in detail.
