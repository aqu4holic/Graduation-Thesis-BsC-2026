Looking at the overall picture, the experiments fall into three groups. The \textit{pure architecture group} -- all attempts to modify the network structure (adding attention branches, dual-path designs, node-centric, etc.) -- either underperforms or matches the baseline. The \textit{feature and data group} -- adding multi-bandwidth kernel channels, ANM residual channels, and X/Y remap augmentation -- improves consistently. % =====================================================================
\section{Architecture Experiment Analysis (Negative Results)}
\label{sec:arch_results}
% =====================================================================

\subsection{v3: Broadcast Scalar Features (5 Channels)}

Version v3 adds partial correlation and residual asymmetry as \textit{constants broadcast} into channels 4 and 5 of the edge tensor (every position in the length-$n=1000$ sequence receives the same scalar value). Results: local drops from 73.96\% to 72.46\% ($-1.50\%$), public drops to 72.13\%.

The reason is clear: Conv1D learns local features along the spatial dimension of sequences. A constant channel with no spatial variation provides nothing for the network to learn, making these channels pure noise in the parameter space. \textit{Lesson: raw statistics cannot be injected as broadcast channels -- they must have spatial variation.}

\subsection{v3+: Observation Cross-Attention Branch}

Version v3+ adds a cross-attention branch over raw observations $(u,v)$: an MLP projects each pair $(u_i, v_i)$ into a token, then cross-attention operates with $Q$ from the conv representation and $KV$ from the observation tokens. Results: 72.06\% local / 71.17\% public -- worse even than v3.

The observation branch adds approximately 100K parameters without appropriate inductive bias: unconstrained attention over 1000 observations has no knowledge of which observations are causally relevant, so it learns noise and overfits. The local-public gap ($0.89\%$) is larger than v3, confirming overfitting.

\subsection{v4: Full Dual-Path Transformer}

Version v4 is the most complex architecture: conv path + observation transformer (2 self-attention layers over 1000 observation tokens + learned query pooling) + bidirectional cross-attention in both directions (conv$\to$obs and obs$\to$conv) + fusion merge. Total approximately 240K parameters. Results: 72.77\% local, but only 68.76\% public -- the largest local-public gap of all experiments ($-4.01\%$ relative to baseline on public), indicating severe overfitting. This experiment had to run on the A100 80GB due to OOM on the A100 40GB with $p=10$ graphs.

\subsection{v5: StatProjector MLP}

Version v5 addresses v3's problem more correctly: partial correlation and residual asymmetry are fed through a dedicated MLP (StatProjector: $2 \to 64 \to 64$) as \textit{per-edge scalars} and injected at the merge step as a third input. Local results: 73.57\% -- close to baseline but still below on public (72.84\%). StatProjector improves the hard classes meaningfully (Mediator $+2.2\%$, Collider $+2.0\%$, Confounder $+1.7\%$) but hurts easier classes, resulting in a net negative globally. \textit{Lesson: statistical features should be injected after Conv1D and global pooling, not at the merge step before self-attention.}

\subsection{v6: Structure-Conditioned Attention}

Version v6 implements a 2-stage design: Stage 1 predicts edge structure (edge classifier) and saves edge embeddings; Stage 2 uses the predicted structure to bias attention -- edges sharing nodes attend more strongly to each other according to the predicted adjacency matrix. Results: 72.28\% local, below baseline. Attention bias conditioned on predicted structure is corrupted by Stage 1 errors, and using the predicted structure to guide attention creates uncontrolled error propagation.

\subsection{v9, v9d: Node-Centric Designs}

v9 and v9d redesign the pipeline from node-centric rather than edge-centric, achieving only 75.43\% and 75.65\% local -- below v8b (76.94\%) despite comparable complexity. Node-centric design reduces the $O(p^2)$ edge context, preventing the model from capturing long-range dependencies between edges in the graph. This confirms that the edge-centric representation of the original solution is the correct architectural choice for this problem.

\subsection{v10: LightGBM Tabular Approach}

Version v10 tests a tabular ML approach (LightGBM) with manual statistical features, achieving only 63.11\% local -- $10.85\%$ below the baseline. This confirms that the end-to-end deep learning approach significantly outperforms manual feature engineering + gradient boosting for this problem, even though rank 3--5 solutions in the competition used XGBoost/LightGBM effectively with very sophisticated feature engineering (923--7000 features)~\cite{olivetti2025adialab}. The key difference lies in the scale of feature engineering: tabular ML needs features designed at a very high level of sophistication to compete with learned deep representations.

\subsection{v13: Full ML Pipeline (Fullstack)}

Version v13 is the most comprehensive ML experiment, implementing a four-stage pipeline: (1) 300+ statistical features per node-pair including Pearson/Spearman/partial correlations, conditional mutual information, HSIC, ANM scores, BIC scores, and regression residuals; (2) classical causal discovery outputs from PC, LiNGAM, NOTEARS, and GES algorithms as additional features; (3) an XGBoost/LightGBM/CatBoost ensemble with X/Y remap augmentation (proper column swap: src$\leftrightarrow$X, tgt$\leftrightarrow$Y); (4) a GNN refinement stage over the predicted graph and a final stacking layer. Despite this substantial engineering effort, v13 achieves only \textbf{72.64\%} local -- marginally below baseline ($-1.32\%$) and far below v11+ ($-8.50\%$).

This result is the most definitive evidence in the thesis for the superiority of the DL approach over the ML approach for this problem. The scalar features extracted in v13 encode the same conditional independence information that the Conv1D learns implicitly from functional curves -- but the scalar representation loses the nonlinear shape information that is critical for distinguishing similar role pairs. Tree-based models operating on fixed feature vectors cannot recover the rich structural patterns captured by Conv1D operating on the full length-1000 sorted-and-regressed sequences. This finding is significant: it establishes that the key information for causal role classification is encoded in the \textit{functional form} of relationships, not merely in aggregate statistical summaries.

\section{Feature and Data Experiment Analysis (Positive Results)}
\label{sec:feature_results}
% =====================================================================

\subsection{v5m: Multi-Bandwidth Kernel Regression (5 Channels)}

Rather than using a single fixed bandwidth $h = 0.5$ as in v2, v5m adds two kernel channels at $h \in \{0.2, 1.0\}$, creating a 5-channel edge tensor (2 sorted + 3 multi-bw kernel). Results: 75.44\% local / 74.80\% public, an increase of $+1.48\%$ over the baseline.

This improvement is meaningful: each bandwidth captures a different smoothness scale of the nonlinear relationship between $u$ and $v$. $h = 0.2$ is sensitive to local structure (sharp slopes, discontinuities), $h = 1.0$ reflects global trends (linear, monotone). The Conv1D model can learn to combine information at multiple spatial scales.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.75\linewidth]{figures/multibw_effect.png}
    \caption{Per-class recall comparison between v2 baseline and v5m (multi-bw kernel). Classes involving causal direction detection (CauseX, CauseY, ConseqX, ConseqY) improve the most.}
    \label{fig:multibw_effect}
\end{figure}

\subsection{v5a: XY Augmentation on v5m Architecture}

v5a applies X/Y remap augmentation (~\ref{sec:xyaug}) on the v5m architecture, expanding from 23,500 original datasets to ${\sim}263$K training samples. Results: 77.91\% local / 77.55\% public, an increase of $+3.95\%$ over baseline and $+2.47\%$ over v5m.

This is the first breakthrough surpassing the official first-place solution (76.70\%) on the public leaderboard. The result confirms that X/Y augmentation is a highly effective technique: it not only increases sample count but also increases the \textit{diversity} of perspectives from which the model observes the same causal structure through different $(X', Y')$ viewpoints.

\subsection{v8b: 8 Channels (5 Kernel + 3 ANM)}

v8b extends v5m by adding 3 multi-bandwidth ANM residual channels $r_h(u_{\sigma(i)}) = v_{\sigma(i)} - \hat{m}_h(u_{\sigma(i)})$ with $h \in \{0.2, 0.5, 1.0\}$, creating an 8-channel edge tensor. Results: 76.94\% local ($+2.98\%$ over baseline).

Improvement concentrates on the two hardest classes: Collider increases $+6.1\%$ and Consequence of X increases $+5.0\%$ compared to v5m alone. This is consistent with ANM theory: the regression residual in the true causal direction should be nearly independent of the source variable, while the reverse direction exhibits stronger nonlinear dependence. The model learns this asymmetric signature from the distributional shape of sequences $\mathbf{c}_{6..8}$.

\subsection{v8b + XY Augmentation}

Combining v8b with XY augmentation achieves \textbf{80.26\%} local / \textbf{80.22\%} public, an increase of $+6.30\%$ over baseline and $+3.32\%$ over v8b alone. This is the first result to exceed 80\% and confirms a synergistic effect between the two techniques: ANM channels improve local causal direction detection, while XY augmentation improves generalization across multiple $(X, Y)$ viewpoints.

\section{Analysis of v11: Structural Attention Bias}
\label{sec:v11_analysis}
% =====================================================================

\subsection{Design and Motivation}

v11 is the sole exception in the architecture group: the only architectural change to deliver significant improvement over the pure baseline. Rather than standard self-attention (treating all edge pairs identically), v11 adds a bias dependent on the \textit{topological relationship type} between two edges in the graph.

For each pair of edges $(u_1, v_1)$ and $(u_2, v_2)$, six topological relationship types $\tau$ are defined: chain ($v_1 = u_2$ or $v_2 = u_1$), fork ($u_1 = u_2$), collider ($v_1 = v_2$), reverse (same pair, opposite direction), sibling, and unrelated. The attention logit is adjusted by a learned scalar bias:
\begin{equation}
  A_{ij} = \frac{q_i \cdot k_j + b_{\tau(i,j)}}{\sqrt{d}},
  \label{eq:v11_bias}
\end{equation}
where $b_\tau \in \mathbb{R}$ is the learned bias for each type $\tau$. The additional parameters are only $|\tau| = 6$ scalars -- negligible -- yet they encode critical topological information: for example, two edges forming a chain $X \to v \to Y$ should be attended to more strongly when classifying Mediator.

The fundamental difference from v3+/v4 is that v11 does not allow attention to learn everything from raw observational data but instead only learns the \textit{degree of priority among topological types}, with a small parameter space and explicit causal prior about graph structure.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.75\linewidth]{figures/v11_structural_bias.png}
    \caption{Structural attention bias in v11: a learned scalar bias indexed by topological relationship type between the edge pair (chain, fork, collider, reverse, unrelated) is added to the attention logit before softmax.}
    \label{fig:v11_bias}
\end{figure}

\subsection{Results}

v11 (base, without augmentation) achieves $79.59\%$ local -- an increase of $+2.65\%$ over v8b ($76.94\%$) and $+5.63\%$ over the baseline. This is the largest improvement from any single architectural change across all experiments.

When combined with XY augmentation, v11+ achieves $\mathbf{81.14\%}$ local and $\mathbf{80.82\%}$ public leaderboard -- the highest result across all experiments, surpassing v8b+ ($80.26\%$ / $80.22\%$) by an additional $+0.88\%$ local and $+0.60\%$ public.

A notable observation is that the marginal gain of XY augmentation on top of v11 ($+1.55\%$) is substantially smaller than on top of v8b ($+3.32\%$). This is because structural bias already encodes topological information about the relationships between edges and the $(X, Y)$ pair directly into the architecture, leaving less new information for augmentation (which provides multiple $(X', Y')$ viewpoints) to contribute.

\subsection{Mediator Error Analysis in v11}

Despite v11's significant improvement, Mediator remains the hardest class. Analysis of 139 Mediator error samples in v11 reveals the following misclassification distribution:
\begin{itemize}
  \item Consequence of X (ConseqX): 45 errors ($32\%$) -- model detects $X \to v$ but misses $v \to Y$
  \item Cause of Y (CauseY): 34 errors ($24\%$) -- model detects $v \to Y$ but misses $X \to v$
  \item Confounder: 30 errors ($22\%$)
\end{itemize}

Approximately $57\%$ of Mediator errors stem from \textit{half-chain detection}: the model identifies one half of the chain $X \to v \to Y$ but fails to identify both. The current chain bias in v11 activates for all edge pairs where $v_1 = u_2$, not specifically for chains passing through both $X$ and $Y$. A promising direction for further improvement is designing X/Y-aware chain bias that activates with higher prior specifically for the chain $X \to v \to Y$.

\section{Synthesis and Lessons Learned}
\label{sec:lessons}
% =====================================================================

\subsection{Causally-Motivated Statistical Features Outperform Architectural Complexity}

Results across all experiments show that \textit{adding statistically-motivated features with clear causal reasoning} (multi-bandwidth kernel regression, ANM residuals) improves performance consistently, while \textit{increasing overall architectural complexity} (adding attention branches, dual-path, node-centric) fails. This is consistent with the analysis in~\cite{olivetti2025adialab}: top solutions in the competition succeeded through ``sophisticated feature engineering and representation learning,'' not through more complex architectures. This thesis provides concrete empirical evidence for that claim.

The v13 ML fullstack experiment provides the strongest version of this evidence: 300+ features, four causal discovery algorithms, three gradient boosting models, a GNN refinement stage, and stacking achieve only 72.64\% -- significantly below the Conv1D baseline at 73.96\%. This demonstrates that what matters is not the quantity of features but the \textit{representational form}: the functional curves captured by sorted+kernel channels contain information about nonlinear causal relationships that cannot be compressed into scalar summaries without loss.

\subsection{Topological Inductive Bias is the Key to Beneficial Architectural Changes}

v11 is an important exception to the lesson above: architecture can help, but \textit{only when} carrying inductive bias appropriate to the problem structure. Structural attention bias works because it encodes explicit causal priors (chain, fork, collider) rather than learning everything from scratch. The difference between v11 ($+2.65\%$ over v8b) and v3+/v4 ($-1.19\%$ to $-1.90\%$) lies not in complexity but in the appropriateness of the inductive bias.

\subsection{Edge-Level $O(p^2)$ Self-Attention is an Irreplaceable Component}

Every attempt to replace or bypass self-attention over the edge set degrades performance. v4 (adding parallel obs transformer) severely overfits due to insufficient inductive bias. v9/v9d (node-centric, reducing $O(p^2)$ edge context) fail to match v8b's performance. This confirms that $O(p^2)$ edge self-attention is the core architectural component enabling global graph reasoning.

\subsection{X/Y Remap Augmentation is the Most Impactful Technical Contribution}

Among all contributions, XY augmentation delivers the largest single improvement ($+3.32\%$ on v8b, $+1.55\%$ on v11). The mechanism: instead of learning to classify the role of $K$ from only one $(X, Y)$ perspective, the model is trained to recognize the same local structure from ${\sim}11$ different perspectives. The result is a more generalizable representation that is less overfit to the specific positions of $X$ and $Y$. A similar technique was independently used by the rank-3 solution in the competition, yielding approximately $+5\%$ improvement and being identified as their primary contribution~\cite{olivetti2025adialab}.

\subsection{Interaction Effect Between Structural Bias and Augmentation}

The marginal gain of XY augmentation on v11 ($+1.55\%$) is substantially smaller than on v8b ($+3.32\%$). The reason: structural bias already encodes topological information about the $X$-$v$-$Y$ relationship directly into the architecture, so augmentation -- which helps the model see multiple $(X', Y')$ viewpoints -- has less new information to contribute. The two techniques are complementary but have partial information overlap -- this is indirect evidence for the mechanism by which structural bias operates.

\subsection{DL vs.\ ML: A Systematic Comparison}

The thesis provides an unusually comprehensive comparison between deep learning and machine learning approaches across multiple experiments (v10 LightGBM: 63.11\%, v13 ML fullstack: 72.64\%, v2 DL baseline: 73.96\%, v11+ best DL: 81.14\%). The conclusion is unambiguous: for this specific problem, deep learning significantly outperforms tabular machine learning regardless of feature engineering sophistication. The Conv1D architecture's ability to learn directly from length-1000 functional curves -- capturing nonlinear shape, curvature, and heteroscedasticity patterns -- provides information that no fixed set of scalar features can recover. This finding has implications for similar causal role classification problems in other domains.

\section*{Chapter 5 Summary}
\addcontentsline{toc}{section}{Chapter 5 Summary}
% =====================================================================

This chapter presented and analyzed the complete experimental results for all 16 model versions developed during the research. Results partition into three clear groups: pure architecture fails; features and data succeed consistently; and v13 ML fullstack definitively confirms DL superiority over manual feature engineering ($72.64\%$ vs $81.14\%$). The v11 structural attention bias is an important exception that reveals the principle underlying successful architectural changes.

The best proposed model -- v11 (8 channels: 5 multi-bandwidth kernel + 3 multi-bandwidth ANM residuals, with topology-aware structural attention bias) combined with X/Y remap augmentation -- achieves $\mathbf{81.14\%}$ Balanced Accuracy on the local test set and $\mathbf{80.82\%}$ on the CrunchDAO public leaderboard, surpassing the competition's first-place solution (\textbf{76.70\%}) by a margin of $+4.12\%$. The four technical contributions (multi-bandwidth kernel, ANM residuals, structural attention bias, XY augmentation) each address specific causes of weakness in the baseline, with particularly strong improvements on Collider and Mediator -- the two hardest classes in the competition.
